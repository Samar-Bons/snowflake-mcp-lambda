Usage Guidelines

Use the spacing system to create consistent layouts and maintain visual hierarchy. Smaller spacing values work well for tight groupings, while larger values help separate distinct sections.

Do

  • Use consistent spacing tokens throughout your design
  • Apply larger spacing between unrelated elements
  • Use smaller spacing for closely related content
  • Stick to the predefined spacing scale

Don't

  • Use arbitrary spacing values outside the scale
  • Apply the same spacing everywhere
  • Ignore the visual hierarchy created by spacing
  • Use extremely large spacing for small components

Code Snippet

/* Using spacing variables */
.element {
  margin: var(--space-4);           /* 16px */
  padding: var(--space-6);          /* 24px */
  margin-bottom: var(--space-8);    /* 32px */
}

Usage Guidelines

Use shadows strategically to create visual hierarchy. Subtle shadows work for most components, while stronger shadows should be reserved for modals, dropdowns, and floating elements.

Do

  • Use consistent shadow levels throughout your interface
  • Apply stronger shadows to elements that float above others
  • Use subtle shadows for cards and panels
  • Consider shadow intensity based on element importance

Don't

  • Use heavy shadows on every element
  • Mix different shadow directions inconsistently
  • Apply shadows to text elements
  • Overuse inner shadows

Usage Guidelines

Icons should enhance usability and provide visual context. Use them consistently and ensure they have appropriate sizing and contrast for accessibility.

Do

  • Use icons to support and clarify actions

Don't

Buttons

Usage Guidelines

Buttons are interactive elements that trigger actions when clicked. Use them to guide users through key workflows and enable important interactions throughout your interface.

Do

  • Use primary buttons for the main call to action
  • Use consistent button sizes within the same context
  • Provide clear, action-oriented labels
  • Use disabled state when actions are temporarily unavailable

Don't

  • Use multiple primary buttons in the same view
  • Use buttons for navigation (use links instead)
  • Make button labels too long or vague
  • Use different button styles inconsistently

Code Snippet

<!-- Primary Button -->
<button class="btn btn--primary">Primary Action</button>

<!-- Secondary Button -->

<!-- Large Button -->
<button class="btn btn--primary btn--large">Large Action</button>

Usage Guidelines

Cards are flexible containers that group related information and actions. They provide a clean way to organize content while maintaining visual hierarchy and scannability.

Do

Don't

  • Mix different card styles inconsistently
  • Make interactive cards without clear hover states

Form Elements

Usage Guidelines

Form elements collect user input and should be designed for clarity, accessibility, and ease of use. Proper labeling and validation help users complete forms successfully.

Do

  • Use clear, descriptive labels for all form fields
  • Provide helpful placeholder text as examples
  • Group related fields logically
  • Show validation feedback clearly

Don't

  • Use placeholder text as a replacement for labels
  • Make forms unnecessarily long or complex
  • Use vague error messages
  • Hide required field indicators

Modals

Usage Guidelines

Modals are overlay windows that focus user attention on specific content or tasks. They should be used sparingly for critical interactions that require user focus or confirmation.

Do

  • Use modals for critical actions that need confirmation
  • Provide clear ways to close the modal (X button, overlay click, escape key)
  • Keep modal content focused and concise
  • Use modals for forms that require user focus

Don't

  • Use modals for non-essential information
  • Chain multiple modals together
  • Make modals too large or complex
  • Open modals automatically without user action

Code Snippet

<!-- Modal Trigger Button -->
<button class="btn btn--primary" onclick="openModal('modal-id')">Open Modal</button>

<!-- Modal Structure -->
<div id="modal-id" class="modal">
    <div class="modal__overlay" onclick="closeModal('modal-id')"></div>
    <div class="modal__content">
        <div class="modal__header">
            <h3 class="modal__title">Modal Title</h3>
            <button class="modal__close" onclick="closeModal('modal-id')">&times;</button>
        </div>
        <div class="modal__body">
            <p>Modal content goes here.</p>
        </div>
        <div class="modal__footer">
            <button class="btn btn--primary">Confirm</button>
            <button class="btn btn--ghost" onclick="closeModal('modal-id')">Cancel</button>
        </div>
    </div>
</div>

Tooltips

Usage Guidelines

Tooltips provide additional context or help text when users hover over elements. They should contain brief, helpful information that clarifies functionality without cluttering the interface.

Do

  • Keep tooltip text short and informative
  • Use tooltips to explain icons or abbreviated text
  • Position tooltips to avoid covering important content
  • Provide keyboard accessibility for tooltips

Don't

  • Use tooltips for essential information
  • Include complex content or multiple lines in tooltips
  • Use tooltips on mobile where hover doesn't exist
  • Cover the element being described

Code Snippet

<!-- Basic Tooltip (bottom) -->

Usage Guidelines

Notifications provide feedback about user actions or system status. They should be timely, relevant, and non-intrusive while clearly communicating the message type and any required actions.

Do

  • Use appropriate notification types for different message severities
  • Include clear, actionable messages
  • Auto-dismiss non-critical notifications
  • Position notifications where they won't block important content

Don't

  • Overuse notifications for minor status updates
  • Use error styling for non-error messages
  • Make critical notifications auto-dismiss too quickly
  • Stack too many notifications at once

Usage Guidelines

Tabs organize related content into separate views within the same context. They help users navigate between different sections while maintaining their current location in the application.

Do

  • Use tabs for content that's at the same level of hierarchy
  • Keep tab labels short and descriptive
  • Limit the number of tabs to avoid overcrowding
  • Clearly indicate which tab is currently active

Don't

  • Use tabs for sequential workflows (use steps instead)
  • Nest tabs within tabs
  • Use tabs for unrelated content
  • Make tabs too small to read or click easily

Code Snippet

<!-- Tab Component -->
<div class="tabs">
